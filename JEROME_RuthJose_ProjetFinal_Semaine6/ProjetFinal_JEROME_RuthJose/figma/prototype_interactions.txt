1. OBJET DU DOCUMENT

Le present document decrit la structure et les interactions du
prototype realise sur Figma. Ce prototype reprend fidelement la
structure validee dans le wireframe version 2 (voir le dossier
wireframe/exports_stitch/) et a servi de reference directe pour le
developpement du site (HTML/CSS/JS).


2. FRAMES REALISEES

  - Accueil (version bureau)
  - Accueil (version mobile)
  - Projets (version bureau)
  - Projets (version mobile)
  - Menu mobile ouvert (overlay)
  - Confirmation d'envoi du formulaire de contact


3. INTERACTIONS DU PROTOTYPE (minimum exige : six)

  1. Clic sur le logo "Ruth-Jose.dev" -> retour a la frame Accueil.

  2. Clic sur le bouton "Voir mes projets" (section Hero) ->
     transition vers la frame Projets.

  3. Clic sur un lien du menu de navigation ("A propos",
     "Competences", "Contact") -> defilement (scroll to) jusqu'a la
     section correspondante de la frame Accueil.

  4. Clic sur l'icone de menu (version mobile) -> ouverture de
     l'overlay "Menu mobile" ; un second clic referme cet overlay.

  5. Clic sur le bouton "Voir" d'une carte projet (frame Accueil)
     -> transition vers la frame Projets, ancree sur le projet
     correspondant.

  6. Clic sur le bouton "Envoyer" du formulaire de contact, une
     fois les champs correctement renseignes -> transition vers la
     frame "Confirmation d'envoi", affichant un message de succes.

  7. Interaction complementaire : survol des boutons principaux ->
     variante visuelle (etat "hover"), permettant d'anticiper le
     retour visuel implemente ensuite dans le site final.


4. COHERENCE ENTRE LE PROTOTYPE ET LE DEVELOPPEMENT

Le site developpe (fichiers site/index.html et site/projects.html)
respecte fidelement la structure et l'enchainement valides dans le
prototype. Certaines ameliorations ont ete integrees au stade du
developpement, sans remettre en cause la logique du prototype :

  - ajout d'un focus clavier visible sur l'ensemble des elements
    interactifs, dans une optique d'accessibilite ;
  - ajout d'un message d'erreur specifique par champ de formulaire,
    plutot qu'un message d'erreur unique et generique ;
  - ajout d'un lien d'evitement ("Aller au contenu principal") pour
    faciliter la navigation au clavier.

Ces ajouts sont documentes de maniere detaillee dans la section
Qualite du fichier README.txt.
